
Baseline v0
개선점: 초기 베이스라인 수립 및 모델별 기본 성능 확인
문제점: 영어와 중국어 혼용, 면접관 역할 이탈(번역기처럼 동작), 질문 개수 통제 불능
알게된 점: 모델 규모에 따른 출력 품질 편차가 매우 크며 기본 안정성 확보가 최우선임
다음 개선사항: 공통 출력 규칙(Common Rules) 도입을 통한 형식 고정
Baseline v1
개선점: 공통 규칙 도입으로 한국어 출력 및 질문 1개 제한 기반 마련
문제점: 소형 모델(7B)의 언어 혼용 잔존, 한 질문 내에 여러 질문을 섞는 다중 질문 발생
알게된 점: 단순 규칙만으로는 모델의 언어 습관을 완벽히 통제하기 어려움
다음 개선사항: 압박형과 친화형 페르소나의 스타일 차별화
Baseline v2
개선점: 압박형(검증 중심)과 친화형(경험 확장 중심)의 역할 정의 및 스타일 규칙 적용
문제점: 두 스타일 간의 실질적 차이 미미, 친화형에서 STAR 구조를 한꺼번에 묻는 장문 발생
알게된 점: 소형 모델은 복잡한 스타일 규칙을 적용할 때 언어 붕괴가 더 빈번함
다음 개선사항: 규칙의 영향력을 확인하기 위한 실험적 구조 개편
Baseline v3 (v3-1 ~ v3-3)
개선점: 질문의 단일성 유지 및 페르소나별 기본 구조 안착
문제점: 첫 질문(FIRST_QUESTION)의 난이도가 신입 수준을 벗어나 너무 높게 생성됨
알게된 점: 공통 규칙은 언어 안정장치일 뿐, 구체적인 행동 제어는 스타일 규칙의 영역임
다음 개선사항: 지원자 수준에 맞는 첫 질문 난이도 정상화
Baseline v4
개선점: 첫 질문 난이도 안정화(탐색형), 다중 질문 형태의 급격한 감소
문제점: 압박형의 구체성 강박 잔존, 스타일이 서로 비슷해지는 수렴 현상 발생
알게된 점: 질문의 스타일보다 면접의 상태(첫 질문/후속 질문) 기반 제어가 더 효과적임
다음 개선사항: 면접 상태(State) 기반의 제어 구조 도입
Baseline v5
개선점: 대부분의 응답이 1문장 1질문으로 고착, 키워드 기반 후속 질문 일관성 향상
문제점: 친화형의 STAR 구조 자동 생성 경향 잔존, 7B 모델의 중국어 혼입 지속
알게된 점: 단순 규칙 추가는 한계가 있으며 구조적 제어나 후처리 검증의 필요성 인지
다음 개선사항: 정량 분석을 위한 스타일 가이드라인 명문화 및 QA 자동 평가 도입
Baseline v6
개선점: 질문의 질보다 규칙 준수율(Compliance) 측정으로 관점 전환, 언어 혼용 빈도 감소
문제점: 가이드라인 준수율이 기대보다 낮음, 압박형이 일반 심화 질문과 구별되지 않음
알게된 점: 규칙 설명보다는 실제 예시(Few-shot)를 제공하는 것이 더 효과적일 것으로 예측됨
다음 개선사항: 퓨샷(Few-shot) 예시 제공 및 규칙 위반 시 재생성 도입
Baseline v7
개선점: 퓨샷 적용으로 스타일 재현성 상승, 압박형과 친화형의 분리 강화
문제점: 모든 답변을 모호하다고 지적하는 과도한 압박, 예시 문장 표현의 반복 노출
알게된 점: 규칙과 퓨샷의 조합이 스타일 유지에 가장 안정적임
다음 개선사항: 재생성(Re-generation) 시스템 도입 및 한국어 필터 적용
Baseline v8
개선점: 키워드 기반 후속 질문 성능 극대화, 압박 질문의 맥락 확인 능력 향상
문제점: 9B 모델의 친화형 장문 폭주 지속, 지원자 답변이 없는데 과거를 묻는 상태 제어 실패
알게된 점: 9B는 과도한 추론이 문제고 7B는 규칙 무시가 문제임(모델별 대응 필요)
다음 개선사항: 자동 재생성 로직 구체화 및 언어 검증기 추가
Baseline v9
개선점: VRAM 최적화로 모델 실행 안정성 확보, 리트라이(Retry) 루프 빈도 감소
문제점: 친화형의 표현 다양성 부족(템플릿화), 첫 질문이 다시 심화되는 경향
알게된 점: 검증 시스템은 필터 역할만 수행할 뿐 실제 출력 방향은 프롬프트가 결정함
다음 개선사항: 페르소나를 텍스트 규칙이 아닌 행동 정의(Action)로 재설계
Baseline v10
개선점: 첫 질문과 후속 질문의 인지적 깊이 차이 명확화, 리트라이 루프 감소
문제점: 친화형의 특정 문구 고착화, 반복되는 주제(리스크 관리 등) 발생
알게된 점: 프롬프트가 설명형일 경우 모델에 강제되지 않으며 다양성이 무너짐
다음 개선사항: 규칙 기반에서 스코어링(Scoring) 기반 평가 시스템으로 전환
Baseline v11 (v11.1 포함)
개선점: 상태 기반 프롬프트가 안정적으로 동작, 압박형 질문의 일관성 고도화
문제점: 리트라이 규칙이 너무 엄격하여 계산 비용 증가, 친화형 장문 문제 해결 안 됨
알게된 점: 프롬프트-검증 정합성이 모델 성능보다 결과에 더 큰 영향을 줌
다음 개선사항: 구조적 제약(Hard Constraint) 시스템 도입
Baseline v12
개선점: 실패 유형의 카테고리화, 프롬프트 개선 단계 종료 및 시스템 아키텍처 확립
문제점: 7B 언어 혼입의 근본적 해결 불가, 친화형의 장문 생성 본능 억제 한계
알게된 점: 프롬프트 튜닝은 한계에 도달했으며 시스템 구조가 성능을 결정함
다음 개선사항: 실패 분석 중심의 구조를 통한 성능 유지 관리
